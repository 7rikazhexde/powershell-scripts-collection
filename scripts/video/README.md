# video（動画変換関連）

ffmpegをフロントエンドにした動画の再エンコード用スクリプトです。

## スクリプト一覧

### [VideoReencoder.ps1](./VideoReencoder.ps1)

ドラッグ＆ドロップ対応の動画再エンコードGUIツール（ffmpegフロントエンド）。
コーデック・pixel format・可変フレームレート（VFR）の不一致による、結合済み動画のカクつき・コマ落ちを解消します。

- 個別に再エンコード（1ファイルずつ）／正規化して1本に結合（複数→1）の2モード
- コーデック（H.264 / H.265）、CRF、プリセット、フレームレート、音声（再エンコード／コピー）を選択可能
- 起動時にffmpegへ実際に1フレーム試しエンコードさせてGPUエンコーダ（NVIDIA NVENC / Intel QSV / AMD AMF）を自動検出し、使用可能なものだけを「エンコーダ」欄に表示（既定は検出できた最速のGPU、なければCPU）
- ドラッグ＆ドロップによる複数ファイル追加、ファイル選択ダイアログからの追加にも対応
- 進捗バー（現在のファイル／全体）とログ表示、処理の中止に対応

```powershell
.\VideoReencoder.ps1
```

## 前提条件

1. ffmpeg（および進捗%表示にはffprobe）が必要です。PATHに通すか、`ffmpeg.exe` / `ffprobe.exe` をこのスクリプトと同じフォルダに置いてください。
2. Windows PowerShell 5.1（STA）での実行を前提としています。PowerShell 7 (pwsh) から起動した場合はWindows PowerShellのSTAプロセスとして自動的に再起動します。
3. 管理者権限では実行しないでください。エクスプローラーからのドラッグ＆ドロップは、昇格したウィンドウに対してWindows（UIPI）によりブロックされます。

## 補足

- GPUエンコードはエンコード側のみで高速化します（デコードはCPU側のまま）。これは異なるソースコーデックが混在していても安定して動作させるための設計です。
- 選択したGPUエンコーダ／コーデックの組み合わせが実行時に利用できない場合は、ログに警告を出したうえでCPUエンコードへ自動的にフォールバックします。
